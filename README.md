# חנות תוספים לאוצריא — גרסה אופליין (Windows)

תוכנה ניידת (portable) עצמאית שמציגה את חנות התוספים של אוצריא (otzaria.org/plugins)
כאפליקציית דסקטופ, עם מסד נתונים מקומי במחשב במקום MongoDB בענן. כל דפי הניהול,
ההעלאה וההתחברות הוסרו — נשארו רק עמודי המשתמש: רשימת תוספים, חיפוש/סינון,
עמוד פרטי תוסף, הורדה, והתקנה ישירה.

נבנתה עם **Tauri** (Rust + ה-WebView שכבר קיים בוינדוז) ולא Electron — קובץ ה-exe
קטן משמעותית (כמה MB בודדים, לעומת עשרות MB בגרסת Electron הישנה).

## איך זה עובד

- **בפעם הראשונה** האפליקציה נפתחת ריקה. לוחצים על **"סנכרון מהאתר"** בפינה
  הימנית העליונה, והאפליקציה מורידה מ־`otzaria.org` את רשימת כל התוספים
  המאושרים, כולל התמונות, צילומי המסך, **וקובץ ה־.otzplugin של כל תוסף**.
- מאותו רגע, החנות עובדת **לגמרי אופליין** — גלישה, חיפוש, צפייה בפרטים והורדת
  קבצים לא דורשים אינטרנט.
- לחיצה נוספת על "סנכרון מהאתר" מרעננת את הרשימה ומוסיפה/מעדכנת תוספים חדשים.
  כדי לחסוך זמן ורוחב פס, קובץ תוסף שהגרסה שלו לא השתנתה לא יורד שוב (התמונות
  וצילומי המסך כן מתעדכנות בכל סנכרון, כי הן קטנות).
- כפתור **"הורדה"** שומר את קובץ ה־.otzplugin (שכבר נמצא במחשב) למיקום לבחירת
  המשתמש (תיבת "שמירה בשם" רגילה של Windows).
- כפתור **"התקנה ישירה"** (מופיע רק לתוספים מסוג `.otzplugin`) פותח
  `otzaria://plugin/install-local?path=<נתיב מקומי מוחלט>` - עובד **לגמרי
  אופליין** (בניגוד ל-`install?url=` הישן שדרש אינטרנט), כי אוצריא קוראת את
  הקובץ ישירות מהדיסק. אם קובץ התוסף עדיין לא סונכרן מקומית, האפליקציה מורידה
  אותו ברקע קודם, כדי שההתקנה הישירה תעבוד תמיד.

## תוכנה ניידת - איפה נשמרים הנתונים

קובץ ה-exe הוא **קובץ יחיד, ללא התקנה**. בכל הרצה, הוא יוצר/משתמש בתיקיית
`plugins-store-data` **תמיד ליד עצמו** (לא בתיקיית משתמש כלשהי במערכת, ולא
בנתיב זמני) - כך שאפשר להעביר את קובץ ה-exe (ואת התיקייה לצידו) לכל מקום,
כולל דיסק-און-קי:

```
<תיקיית ה-exe>\
  otzaria-plugins-store.exe
  plugins-store-data\
    db.json          <- מטא-דאטה על כל התוספים (שם, גרסה, תגיות, נתיבי קבצים...)
    files\<id>\...    <- התמונה, צילומי המסך וקובץ ה־.otzplugin של כל תוסף
```

מחיקת התיקייה הזו שקולה ל"איפוס" מלא של האפליקציה (יידרש סנכרון מחדש).

## מבנה הפרויקט

```
src/                  frontend - HTML/CSS/JS פשוטים, בלי בנייה/bundler
  index.html
  app.js
  styles.css
  tauri-bridge.js      גשר בין ה-frontend לבין ה-backend של Tauri
  assets/logo.svg
src-tauri/             backend ב-Rust
  src/main.rs           כל הלוגיקה: סנכרון, הורדות, התקנה ישירה, זיהוי תוספים מותקנים
  icons/icon.ico         אייקון האפליקציה
  tauri.conf.json
  Cargo.toml
```

## הרצה בזמן פיתוח / בנייה

דורש [Rust](https://rustup.rs) ו-Tauri CLI (`cargo install tauri-cli --version "^2"`).
פרטים מלאים, כולל דרישות מקדימות בוינדוז, ב-[tauri.app](https://tauri.app).

```bash
# הרצה מקומית לבדיקה (עם hot-reload):
cargo tauri dev

# בנייה סופית - קובץ exe יחיד תחת src-tauri/target/release/:
cargo tauri build
```

**בנייה אוטומטית:** בכל push ל-`main` שנוגע ב-`src/` או `src-tauri/`,
GitHub Actions בונה את האפליקציה על `windows-latest`, מקדמת אוטומטית את מספר
הגרסה (patch), ומפרסמת **GitHub Release** חדש עם קובץ ה-exe מצורף (טאב
Releases של הריפו).

## מה הוסר לעומת האתר המקורי

- כל מערכת ההתחברות/הרשמה (`next-auth`), עריכת תוספים, העלאת תוסף חדש, וכל
  עמודי הניהול (`/library/admin/*`, אישור תוספים, סטטיסטיקות וכו').
- מערכת ה"ספרייה" הכללית של אוצריא (OCR, עריכת ספרים, דיקטה וכו') — זו לא
  חלק מחנות התוספים ולא הייתה חלק מהבקשה.
- ה-endpoint הפנימי `/api/plugins/resolve` (משמש רק את ה-GitHub Action
  לפרסום אוטומטי של תוספים — לא רלוונטי לאפליקציית משתמש קצה).

## דברים לשים לב אליהם

- הסנכרון הראשון עלול לקחת זמן אם יש הרבה תוספים/קבצים גדולים — יש בעיה
  (bar) עם התקדמות ורשימת אזהרות אם קובץ מסוים נכשל (התוסף פשוט יישאר בלי
  אותו קובץ ספציפי, וניתן לנסות סנכרון נוסף מאוחר יותר).
- האפליקציה קוראת בכל סנכרון את כתובת `https://otzaria.org/api/plugins` —
  אם כתובת האתר תשתנה בעתיד, יש לעדכן את הקבוע `BASE_URL` בתחילת
  `src-tauri/src/main.rs`.
