# חנות תוספים לאוצריא — גרסה אופליין (Windows, Tauri)

אפליקציית דסקטופ עצמאית (Rust + Tauri, ללא Electron) שמציגה את חנות התוספים
של אוצריא (otzaria.org/plugins) עם מסד נתונים מקומי במחשב במקום MongoDB
בענן. כל דפי הניהול, ההעלאה וההתחברות הוסרו — נשארו רק עמודי המשתמש: רשימת
תוספים, חיפוש/סינון, עמוד פרטי תוסף, הורדה, והתקנה ישירה.

בזכות Tauri (WebView2 שכבר מותקן בוינדוז, במקום Chromium+Node ארוזים בפנים),
קובץ ה-exe הסופי קטן משמעותית מהגרסה הישנה שהתבססה על Electron (כ-5-15MB
במקום עשרות MB).

## איך זה עובד

- **בפעם הראשונה** האפליקציה נפתחת ריקה. לוחצים על **"סנכרון מהאתר"** בפינה
  הימנית העליונה, והאפליקציה מורידה מ־`otzaria.org` את רשימת כל התוספים
  המאושרים, כולל התמונות, צילומי המסך, **וקובץ ה־.otzplugin של כל תוסף**.
- מאותו רגע, החנות עובדת **לגמרי אופליין** — גלישה, חיפוש, צפייה בפרטים והורדת
  קבצים לא דורשים אינטרנט.
- לחיצה נוספת על "סנכרון מהאתר" מרעננת את הרשימה ומוסיפה/מעדכנת תוספים חדשים.
  כדי לחסוך זמן ורוחב פס, קובץ תוסף שהגרסה שלו לא השתנתה לא יורד שוב (התמונות
  וצילומי המסך כן מתעדכנים בכל סנכרון, כי הם קטנים).
- כפתור **"הורדה"** שומר את קובץ ה־.otzplugin (שכבר נמצא במחשב) למיקום לבחירת
  המשתמש (תיבת "שמירה בשם" רגילה של Windows).
- כפתור **"התקנה ישירה"** (מופיע רק לתוספים מסוג `.otzplugin`) פותח קישור
  `otzaria://plugin/install-local?...` — זה עובד רק אם תוכנת אוצריא הראשית
  מותקנת במחשב ורשומה כמטפלת בפרוטוקול הזה.

## מבנה הנתונים המקומי

התוכנה ניידת (portable) לגמרי: הנתונים (מסד הנתונים + כל הקבצים שהורדו)
נשמרים בתיקייה **צמודה לקובץ ה-exe עצמו**, לא בתיקיית משתמש כלשהי במערכת:

```
<תיקיית ה-exe>/plugins-store-data/
  db.json          <- מטא-דאטה על כל התוספים (שם, גרסה, תגיות, נתיבי קבצים...)
  files/<id>/...    <- התמונה, צילומי המסך וקובץ ה־.otzplugin של כל תוסף
```

כך שהעתקת קובץ ה-exe (למשל לדיסק-און-קי) מעבירה גם את הנתונים. מחיקת התיקייה
הזו שקולה ל"איפוס" מלא של האפליקציה (יידרש סנכרון מחדש).

## מבנה הפרויקט

```
.
├── assets/logo.svg              (מקור הלוגו, ממנו מופקים כל האייקונים)
├── src/                         (frontend - HTML/JS/CSS פשוטים, בלי build step)
│   ├── index.html
│   ├── app.js
│   ├── styles.css
│   └── tauri-bridge.js          (חושף window.api על גבי Tauri commands/events)
└── src-tauri/
    ├── Cargo.toml                (תלויות Rust)
    ├── build.rs
    ├── tauri.conf.json           (הגדרות אפליקציה/חלון/bundle)
    ├── capabilities/default.json (הרשאות)
    └── src/main.rs                (כל לוגיקת ה-backend: סנכרון, הורדות, זיהוי תוספים מותקנים וכו')
```

## התקנות מקדימות (פעם אחת בלבד, בוינדוז)

1. **Rust**: התקן דרך https://rustup.rs
2. **Visual Studio Build Tools** (נדרש ל-Rust בוינדוז, אם `rustup` לא מתקין
   אוטומטית): https://visualstudio.microsoft.com/visual-cpp-build-tools —
   לבחור "Desktop development with C++"
3. **WebView2**: כבר מותקן כברירת מחדל בוינדוז 10/11 עדכניים; אם לא —
   https://developer.microsoft.com/microsoft-edge/webview2
4. **Tauri CLI**:
   ```bash
   cargo install tauri-cli --version "^2"
   ```

## הרצה בזמן פיתוח

```bash
# שלב חד-פעמי: יוצר אוטומטית את כל קבצי האייקונים (icon.ico וכו') מתוך הלוגו הקיים
cargo tauri icon assets/logo.svg

# הרצה מקומית עם hot-reload:
cargo tauri dev
```

## בנייה

```bash
cargo tauri build
```

התוצר הוא קובץ יחיד: `src-tauri/target/release/otzaria-plugins-store.exe`.
אין installer בכלל (מבוטל בכוונה ב-`tauri.conf.json` דרך `"bundle": {"active": false}`)
— זו תוכנה ניידת (portable) לגמרי.

## מה הוסר לעומת האתר המקורי

- כל מערכת ההתחברות/הרשמה (`next-auth`), עריכת תוספים, העלאת תוסף חדש, וכל
  עמודי הניהול (`/library/admin/*`, אישור תוספים, סטטיסטיקות וכו').
- מערכת ה"ספרייה" הכללית של אוצריא (OCR, עריכת ספרים, דיקטה וכו') — זו לא
  חלק מחנות התוספים.
- ה-endpoint הפנימי `/api/plugins/resolve` (משמש רק את ה-GitHub Action
  לפרסום אוטומטי של תוספים — לא רלוונטי לאפליקציית משתמש קצה).

## דברים לשים לב אליהם

- הסנכרון הראשון עלול לקחת זמן אם יש הרבה תוספים/קבצים גדולים — יש בעיה
  (bar) עם התקדמות ורשימת אזהרות אם קובץ מסוים נכשל (התוסף פשוט יישאר בלי
  אותו קובץ ספציפי, וניתן לנסות סנכרון נוסף מאוחר יותר).
- האפליקציה קוראת בכל סנכרון את כתובת `https://otzaria.org/api/plugins` —
  אם כתובת האתר תשתנה בעתיד, יש לעדכן את הקבוע `BASE_URL` בתחילת
  `src-tauri/src/main.rs`.
- הקוד ב-`main.rs` נכתב לפי הידע על ה-API של Tauri v2 אך לא עבר קימפול מלא
  בסביבת הפיתוח שבה נכתב (ראו הערה בראש הקובץ). אם יעלו שגיאות קימפול
  ראשוניות בוינדוז, אלה כמעט תמיד תיקוני שם-מתודה/טיפוס קטנים סביב
  `tauri-plugin-dialog`, לא בעיה בלוגיקה עצמה.
